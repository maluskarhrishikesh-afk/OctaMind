# OctaMind — Product Requirements

> **Last updated:** 2026-02-21
> **Status:** Living document — append new requirements here; do not alter historical entries.

---

## 1. Brand & Identity

- Product name: **OctaMind** — derived from the 8 legs of an octopus, representing the ability to run
  multiple agents simultaneously.
- Logo: `src/agent/assets/octopus.png` — a pink octopus displayed in every agent window and the
  dashboard header.
- Primary accent colour: **#e91e8c** (hot-pink).

---

## 2. Agent Hub Dashboard

- A central dashboard (port 8501) where all agents are listed and managed.
- Supported agent types at launch:
  - **Gmail** — email management
  - **Google Drive** — file management  *(planned)*
  - **Slack** — messaging  *(planned)*
  - **Calendar** — scheduling  *(planned)*
  - **Stock Market** — market data  *(planned)*
  - **Custom** — user-defined purpose
- The dashboard must keep running until the browser window is closed.
- Creating an agent on the dashboard must **not** interrupt other running agents.

---

## 3. Agent Creation

- Users can create multiple agents via a form.
- Required fields: **Name**, **Agent Type**, **Role / Purpose**.
- Optional fields:
  - **Personality Traits** — five sliders (Tone, Verbosity, Humor, Empathy, Proactiveness) on
    a 0–10 scale. Defaults are pre-filled; the user may customise them.
  - **Advanced Configuration** — auto-run on startup, max operations per command.
- On creation the agent is allocated its own memory folder (`memory/<agent_id>/`) and a structured
  `personality.md` is generated from the trait sliders.

---

## 4. Agent Memory

Each agent must have a persistent, multi-layer cognitive memory:

| Layer               | File                | Purpose                                 |
| ------------------- | ------------------- | --------------------------------------- |
| Working memory      | `working_memory.md` | Last 10 interactions (RAM-like, trimmed)|
| Episodic memory     | `episodic_memory.md`| Time-stamped events with importance     |
| Semantic memory     | `semantic_memory.md`| Extracted facts about the user          |
| Personality         | `personality.md`    | Agent identity + trait scores           |
| Habits              | `habits.md`         | Confirmed patterns (3+ occurrences)     |
| Consciousness       | `consciousness.md`  | Meta-summary layer (updated monthly)    |

- Full specification in `requirement/memory_architecture.md`.
- Architecture must match `architecture/memory-system.md` at all times.
- When asked "Do you know what we did earlier?", the agent must recall relevant history.
- The interaction should feel like talking to a real human assistant.

---

## 5. Agent Lifecycle

- **Start Agent** — opens the agent in its own Streamlit window (dedicated port).  The dashboard
  remains visible and functional during and after the start.
- **Stop Agent** — shuts down the agent process.
- **Two stop methods:**
  1. Clicking **Stop Agent** in the dashboard.
  2. Closing the agent browser window / tab.
- The agent always displays its own name in the window header and refers to itself by that name.
- Users chat with the agent via a chat interface inside the agent window.

---

## 6. Configure Agent (Post-Creation)

The ⚙️ Configure button on each agent card opens an inline panel with:

### 6a. Personality Traits (editable)
- Same five sliders as during creation.
- Saving rewrites `personality.md` immediately.

### 6b. Automations (per agent type)
- A list of recurring background tasks that run while the agent process is active.
- Each automation has an **Enable / Disable toggle**; unchecking it stops the task immediately.
- Enabled automations run on a background daemon thread (60-second scheduler tick).
- State persisted in `memory/<agent_id>/automation_config.json`.
- **Gmail automations (10 built-in):**
  1. 🗑 Auto-delete spam (every 15 min)
  2. 📁 Auto-archive newsletters (every 30 min) — configurable label
  3. 📋 Daily email digest (every 24 h)
  4. ⭐ Auto-label VIP emails (every 15 min)
  5. 🚩 Flag old unread emails (every 24 h) — configurable age threshold
  6. 📊 Weekly productivity report (every 7 days)
  7. 🗂 Auto-categorize by sender domain (every 30 min)
  8. 🔕 Detect promotional senders (every 7 days) — configurable confidence threshold
  9. 💬 Auto-reply / Out of Office (every 15 min) — configurable message & end date
  10. 🧹 Archive old read emails (every 24 h) — configurable age threshold
- Future agent types (Drive, Slack, etc.) will register their own automation catalogs without
  changing the framework.

## 7. Executables

- `start.exe` — double-click to launch OctaMind; detects if already running; opens browser.
- `stop.exe`  — double-click to stop all agents and the dashboard.
- Both compiled with **PyInstaller `--onefile`** using `octopus.ico`.
- Source scripts: `start.py` / `stop.py` at project root.
