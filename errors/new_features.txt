2. Memory Architecture (Your 6 Layers)

Your assistant has 6 memory layers.

1. Working Memory
2. Semantic Memory
3. Episodic Memory
4. Habits
5. User Personality
6. Self Reflection

Plus two operational layers:

Context
Manifest
3. Role of Each Memory Layer
1️⃣ Working Memory

Short-term memory for the current task.

Example:

task: organize_images
files_found: 482
folder: /home/downloads

Location:

memory/working/

This always goes to the LLM.

2️⃣ Semantic Memory

Stores facts about the user and environment.

Example:

User prefers Downloads folder
User uses Google Drive
User uses Linux

Location:

memory/semantic/

Format:

---
tags: file_organization
importance: high
---

User prefers files to be organized inside the Downloads folder.

Retrieval method:

Embedding similarity search

3️⃣ Episodic Memory

Stores past experiences.

Example:

User organized PDFs last week using year folders.

Location:

memory/episodic/

Example:

---
task: file_organization
date: 2026-03-04
---

User organized PDFs into year folders.
User liked the result.

Retrieval method:

Embedding similarity search

4️⃣ Habits

Stores patterns in user behaviour.

Example:

User prefers:
Year → Month folder structure

Location:

memory/habits/

Example:

habit: folder_structure
pattern: year/month
confidence: 0.87

Retrieval method:

Rule-based

5️⃣ User Personality

Stores communication and work style of the user.

Example:

communication_style: concise
technical_level: high
organization_style: structured

Location:

memory/personality/

Example:

communication: concise
technical_level: high
decision_style: data_driven
assistant_proactivity: medium

This influences assistant behaviour.

6️⃣ Self Reflection

Stores lessons learned from past mistakes.

Example:

strategy_used: type_based
result: rejected
better_strategy: year_based

Location:

memory/self_reflection/

Example:

task_type: file_organization

strategy_used: type_based
user_feedback: negative

improved_strategy: year_based

Retrieval method:

Task-based retrieval

4. Manifest (Task Execution State)

Manifest tracks long-running tasks.

Example:

task_id: 9217

steps:
scan_files: done
classify_files: done
move_files: running

Location:

manifest/tasks.json

This allows:

✔ task recovery
✔ async execution
✔ background tasks

The LLM usually does NOT see this.

5. Context vs Memory (Very Important)

Many systems fail here.

Concept	Meaning
Memory	Stored on disk
Context	Sent to LLM

Example:

Total memory: 5 GB
LLM context: 2000 tokens

You only send relevant memory.

6. Memory Retrieval Strategy

When a user request arrives:

Step 1 — Detect task

Example:

organize images
Step 2 — Retrieve memory

Memory manager fetches:

Working memory
Top 3 semantic memories
Top 3 episodic memories
Relevant habits
User personality
Relevant self reflection
Step 3 — Build LLM context

Example:

User request:
Organize my images.

User personality:
Communication style: concise

User habits:
Prefers year/month folder structure.

Relevant past experience:
User organized PDFs using year folders.

Self reflection:
Type based organization was rejected.

Task context:
482 images detected.

Then send to LLM.

7. How Cosine Similarity Works With MD Files

Your memory files remain plain markdown.

Example folder:

memory/
  semantic/
  episodic/
  habits/
  personality/
  self_reflection/

Process:

MD Files
  |
Read Files
  |
Generate Embeddings
  |
Store Vector Index
  |
Cosine Similarity Search

Tools you can use:

Tool	Purpose
SentenceTransformers	generate embeddings
FAISS	fast vector search
Chroma	simple vector DB

Example flow:

query: organize images

↓ embedding

vector search

↓ top matches

episodic/organized_pdfs.md
habits/folder_structure.md
semantic/downloads_preference.md

13. Context Control Rule (Critical)

Never send:

all episodic memory
all semantic memory

Instead send:

Top K relevant memories

Typical values:

semantic: 3
episodic: 3
habits: 1-2
self_reflection: 1

This keeps context under control.
